## Project Overview

End-to-end natural disaster prediction system combining ESP32 IoT firmware with a Python ML pipeline and a Streamlit dashboard. The ESP32 simulates environmental sensors (temperature, humidity, luminosity, water level, soil vibration) and streams labeled sensor data via serial port. A Jupyter notebook trains a Random Forest classifier to predict 7 disaster classes (0 = safe, 1–6 = heat/humidity, fire/smoke, moderate vibration, landslide, high water level, flooding). A Streamlit dashboard loads the exported model and provides real-time monitoring and prediction.

**Authors:** Ian Nobres Azevedo and Matheus Mikio Tutume Lourenço

## Architecture

```
ESP32 serial CSV (admin mode) → Excel → Notebook ML → disaster_model.pkl → Dashboard
```

### Part 1 — ESP32 Firmware (`src/`)
- Written in Arduino/C++ for ESP32Dev board
- Constants and thresholds in `src/config.h` (edit thresholds here, not in .ino)
- Non-blocking state machine: `NORMAL → AGUARDANDO_SENHA → BLOQUEADO → ADMIN`
- **Normal mode:** reads sensors, validates NaN/div-by-zero, outputs compact CSV with boolean flags. Green/yellow/red LEDs reflect current status.
- **Admin mode:** SHA256-authenticated (password is `ADMINGS`); outputs full CSV with timestamp, ID, raw sensor values, risk text, and multiclass ML label (0–6). Auto-timeout after 5 minutes of inactivity.

**Normal mode serial columns:**
`Temperatura, Umidade, Luminosidade (bool), Nivel da agua (bool), Vibração do solo (bool), Status Risco`

**Admin mode serial columns:**
`Data, ID, Temperatura, Umidade, Luminosidade (lux), Nivel da agua (cm), Vibração do solo (ADC), Status Risco, Saída ML`

`Saída ML` now outputs the 7-class label (0–6) directly from the firmware, aligned with the ML model classes.

- Sensor pins: DHT22 on pin 16, LDR on pin 4, ultrasonic (TRIG/ECHO) on pins 5/18, potentiometer (vibration proxy) on pin 27

### Part 2 — ML Pipeline (`ml/`)
- Loads `ml/data/Dados ESP32 GS.xlsx` (2457 rows, 9 columns — 52 simulated days)
- Feature engineering: derives `Dia`, `Hora`, `Minuto`, `Tempo_Minutos`, `Hora_sin`, `Hora_cos` from timestamp
- Random Forest Classifier with `class_weight='balanced'` (handles class imbalance, especially class 5 with ~16 samples)
- 5-Fold StratifiedKFold cross-validation reporting F1-Macro mean ± std
- Feature importance plot and false negative analysis
- Exports trained model to `ml/models/disaster_model.pkl` via joblib
- Achieves ~85% test accuracy

### Part 3 — Dashboard (`dashboard/`)
- Streamlit app with 3 tabs:
  - **Monitor:** time-series charts of all sensors, key metrics
  - **Previsão:** upload CSV or manual slider input → loads model → shows prediction + class probabilities
  - **Insights:** feature importance, class distribution, model parameters
- Loads model from `ml/models/disaster_model.pkl` (run notebook first)
- Demo data auto-loaded from `ml/data/Dados ESP32 GS.xlsx`

## Commands

### Firmware (PlatformIO)

```bash
# Build firmware
pio run

# Upload to connected ESP32
pio run --target upload

# Monitor serial output (115200 baud)
pio device monitor --baud 115200

# Clean build artifacts
pio run --target clean
```

### ML Notebook

```bash
# Install Python dependencies
cd ml
pip install -r requirements.txt

# Launch notebook
jupyter notebook "Modelo_Previsão_de_Desastre.ipynb"
# Run all cells — generates ml/models/disaster_model.pkl
```

### Dashboard

```bash
cd dashboard
pip install -r requirements.txt
streamlit run app.py
# Open http://localhost:8501
```

### Wokwi Simulation

Build firmware with `pio run`, then open `wokwi.toml` in VS Code with the Wokwi extension. The `.toml` points to `.pio/build/esp32/firmware.elf` and `.bin`.

## Key Files

| File | Purpose |
|---|---|
| `src/ProjetoGS.ino` | ESP32 firmware — state machine, sensors, LEDs, serial output |

| `ml/Modelo_Previsão_de_Desastre.ipynb` | ML pipeline: EDA → training → evaluation → export |

| `wokwi.toml` | Points Wokwi extension to built ELF/BIN artifacts |

## ML Label Classes

| Class | Description |
|---|---|
| 0 | Safe — no disaster |
| 1 | Fire/heat alert (moderate conditions) |
| 2 | Fire or dense smoke (critical luminosity) |
| 3 | Moderate vibration alert |
| 4 | Landslide (strong vibration) |
| 5 | Flood alert (elevated water level) |
| 6 | Imminent flooding |
